2) Создание меню Файл. Диалоги открытия и сохранения файлов

Сначала выполним подготовительную работу.

Используя инструмент Ножницы, вырежьте и сохраните в png-файлах иконки для некоторых операций:

-Open.png, -Save.png, -Cut.png, -Copy.png, -Paste.png, -Undo.png, -Redo.png.

 

Для создания меню разместите на форме элемент MenuStrip1. При этом сам элемент разместится внизу на желтой панели, а на форме появится приглашающий текст Вводить здесь.

Введите название меню – Файл.

А ниже введите два пункта меню: Открыть и Сохранить.

Для пункта Открыть в качестве значения свойства Image выберите файл -Open.png, а для пункта Сохранить – файл -Save.png.

Запустите приложение, убедитесь в наличии меню.

Закройте приложение, вернитесь к редактированию.

Разместите на форме два диалоговых окна: OpenFileDialog1 и SaveFileDialog1 (они также окажутся внизу на желтой панели). Эти элементы обеспечивают вызов стандартных диалогов открытия и, соответственно, сохранения файлов.

 

Зададим тип файлов, которые будут отображаться в этих диалогах. Для того и другого диалога задайте значение свойства:

Filter = Документы (*.doc)|*.doc|Все файлы (*.*)|*.*

Для обоих диалогов задайте также исходную папку – пусть это будет папка нашего проекта:

InitialDirectory = C:\PABCWork.NET\310\Editor

Можно также задать расширение файла, которое, если пользователь не укажет, будет автоматически приписываться к имени файла:

DefaultExt = doc

 

Теперь свяжем пункты меню Открыть и Сохранить с соответствующими диалогами.

Двойным щелчком по пункту Открыть создайте процедуру обработки выбора этого пункта и впишите туда два оператора:

procedure Form1.toolStripMenuItem3_Click(sender: Object; e: EventArgs);

begin

  openFileDialog1.ShowDialog;

  richTextBox1.LoadFile(openFileDialog1.FileName);

end;

openFileDialog1.ShowDialog – так активизируется диалог открытия файла.

richTextBox1.LoadFile(openFileDialog1.FileName) – а так загружается содержимое выбранного файла в редактируемое текстовое поле.

 

Двойным щелчком по пункту Сохранить создайте процедуру обработки выбора этого пункта и впишите туда два оператора:

procedure Form1.toolStripMenuItem4_Click(sender: Object; e: EventArgs);

begin

  saveFileDialog1.ShowDialog;

  richTextBox1.saveFile(saveFileDialog1.FileName);

end;

saveFileDialog1.ShowDialog – открытие диалога по сохранению файла.

richTextBox1.saveFile(saveFileDialog1.FileName) – содержимое текстового поля сохраняется в указанном в диалоге файле; если такой файл уже есть – будет предложено его заменить , если такого файла нет – он будет создан.

 

Запустите приложение и протестируйте его.

Введите какой-нибудь текст и сохраните его в файле под именем, например, MyDocument.doc.

Затем очистите редактируемое поле и откройте файл MyDocument.doc.

Убедитесь: загрузка файла произошла.

Закройте приложение, вернитесь к редактированию.

 

Примечание. Вместо текстового поля типа richTextBox можно использовать поле типа textBox (со свойством MultiLine = True).

В отличие от richTextBox у поля textBox отсутствуют методы SaveFile и LoadFile.

Сохранить содержимое поля textBox1.Text в файле с именем NFile можно так:

System.IO.File.WriteAllText(NFile, textBox11.Text);

А, наоборот, прочитать файл и поместить его содержимое в поле можно так:

textBox1.Text := System.IO.File.ReadAllText(NFile);

Имя файла NFile можно брать как свойство диалогов:

NFile := openFileDialog1.FileName;

NFile := saveFileDialog1.FileName;

 

3) Создание меню Правка. Работа с буфером обмена

Правее меню Файл добавьте меню Правка с тремя пунктами: -Вырезать, -Копировать, -Вставить.

Для каждого пункта в свойство Image загрузите соответствующие иконки.

 

Для п. Вырезать создайте процедуру-обработчик выбора этого пункта:

procedure Form1.toolStripMenuItem6_Click(sender: Object; e: EventArgs);

begin

  richTextBox1.Cut;

end;

Для п. Копировать:

procedure Form1.toolStripMenuItem7_Click(sender: Object; e: EventArgs);

begin

    richTextBox1.Copy;

end;

Для п. Вставить:

procedure Form1.toolStripMenuItem8_Click(sender: Object; e: EventArgs);

begin

  richTextBox1.Paste;

end;

– во всех этих трех процедурах используются соответствующие методы текстового поля: Cut, Copy, Paste.

Процедуры обработки выбора этих пунктов должны содержать операторы:

4) Создание меню Текст. Диалоги задания параметров шрифта

Разместите на форме еще два диалога: fontDialog и colorDialog, – которые служат для вызова стандартных диалоговых окон для задания параметров шрифта и, соответственно, цвета шрифта.

 

Правее меню Правка добавьте меню Текст с двумя пунктами: Шрифт и Цвет.

procedure Form1.toolStripMenuItem12_Click(sender: Object; e: EventArgs);

begin

  fontDialog1.ShowDialog;

end;

rocedure Form1.toolStripMenuItem13_Click(sender: Object; e: EventArgs);

В этих процедурах сначала активизируется диалог, а потом заданные в диалоге параметры применяются к предварительно выделенному фрагменту текста.

 

Запустите и протестируйте приложение.

5) Создание контекстного меню

Разместите на форме элемент контекстного меню contextMenuStrip1. При этом сам элемент отобразится на желтой панели внизу, а на форме появится приглашающий текст Вводить здесь.

Введите два пункта: Шрифт и Цвет.

Тело процедуры для п. Шрифт:

  fontDialog1.ShowDialog;

  richTextBox1.SelectionFont := fontDialog1.Font;

Тело процедуры для п. Цвет:

  colorDialog1.ShowDialog;

  richTextBox1.SelectionColor := ColorDialog1.Color;

– сначала активизируется соответствующий диалог, а потом заданные в диалоге параметры применяются к предварительно выделенному фрагменту текста (так же, как и в процедурах обработки пунктов меню Текст).

 

Текстовому полю надо сообщить, что у него есть контекстное меню. Для этого на форме выберите элемент текстового поля richTextBox1 и в его свойстве ContextMenuStrip выберите contextMenuStrip1.

 

Запустите приложение. И убедитесь, что через контекстное меню предварительно выделенного фрагмента текста можно задавать параметры и цвет текста.

Закройте приложение, вернитесь к редактированию.


6) Создание панели инструментов

Создадим панель инструментов и разместим на ней несколько кнопок для выполнения некоторых действий.

Сначала уменьшим размер текстового поля, освободим место под панель инструментов. Для этого выделите текстовое поле и задайте для него свойство Dock = Bottom.

Теперь разместите на форме элемент управления toolStrip1 – панель инструментов.

В области панели инструментов появится кнопка ниспадающего меню со списком типов элементов. Выберите тип Button – кнопка. Эта кнопка будет открывать файл (дублируя действие пункта меню Файл).

В качестве значения свойства Image (иконка для этой кнопки) загрузите файл -Open.png.

Двойным щелчком по кнопке создайте процедуру обработки клика и в тело процедуры впишите знакомые нам уже операторы:

procedure Form1.toolStripButton1_Click(sender: Object; e: EventArgs);

begin

  openFileDialog1.ShowDialog;

  richTextBox1.LoadFile(openFileDialog1.FileName); 

end;

 

Добавьте на панель инструментов еще одну кнопку Button – с иконкой из файла -Save.png и с процедурой:

procedure Form1.toolStripButton2_Click(sender: Object; e: EventArgs);

begin

  saveFileDialog1.ShowDialog;

  richTextBox1.saveFile(saveFileDialog1.FileName);

end;

Эта кнопка будет служить для сохранения файла.

 

Запустите приложение и убедитесь, что кнопки панели инструментов вызывают соответствующие диалоги: открывают и сохраняют файл.

Закройте приложение, вернитесь к редактированию.

 

Создадим еще две кнопки, которые позволят задавать набор параметров шрифта: одна кнопка – для заголовков, а другая – для обычного текста.

 

Сначала разместите на панели инструментов Separator – разделительную линию.

 

Потом добавьте кнопку Button. И чтобы она содержала не иконку, а текст, задайте для неё значение свойства DisplayStyle = Text (вместо Image), задайте значение свойства Text = Заголовок.

В свойстве Font щелкните по кнопке … – появится диалог Шрифт. Те параметры, которые мы здесь установим, будут присваиваться форматируемому тексту, как заголовку. Задайте размер – 12 и начертание – полужирный. Ok.

Процедура:

procedure Form1.toolStripButton3_Click(sender: Object; e: EventArgs);

begin

  richTextBox1.SelectionFont := toolStripButton3.Font;

end;

– параметры шрифта кнопки панели инструментов будут присвоены текущему тексту в текстовом поле. (У меня номер этой кнопки – 3, у вас, возможно, другой).

 

Добавьте на панель инструментов еще одну кнопку Button – для обычного текста; свойства кнопки: DisplayStyle = Text, Text = Обычный. Параметры свойства Font пусть останутся какие есть, они будут присваиваться обычному тексту.

Процедура:

procedure Form1.toolStripButton4_Click(sender: Object; e: EventArgs);

begin

  richTextBox1.SelectionFont := toolStripButton4.Font;

end;

 

Запустите приложение и убедитесь в действенности кнопок панели инструментов для форматирования заголовков и обычного текста.

Закройте приложение.
